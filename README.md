🏥 Smart Healthcare AI — Intelligent Triage & Decision Support System

Built for a Hackathon | B.Tech Final Year Project | 2025–26

An end-to-end AI-powered hospital triage system that analyzes patient vitals in real time, predicts severity using a trained ML model, and routes patients to the right doctor — all in under 2 seconds.

🎯 What It Does
Hospitals waste critical minutes manually triaging patients. This system automates that:

Nurse enters patient vitals
AI predicts severity — GREEN / YELLOW / ORANGE / RED
System auto-assigns the right specialist doctor
Doctor reviews patients on a priority dashboard
Patient checks their own status and receives doctor instructions


🤖 AI Model
MetricScoreModelGradient Boosting ClassifierAccuracy~95%ROC-AUC0.97Training Data2000 patient recordsFeatures29 engineered features
Key features used: Heart Rate, SpO₂, Blood Pressure, Temperature, Respiratory Rate, Chest Pain, Diabetes, Heart Disease history + engineered ratios (Shock Index, HR/Oxygen ratio, Danger Score, etc.)

⚙️ Setup & Run
Prerequisites

Python 3.10+
MySQL running locally
MySQL database named ai_triage_db

Step 1 — Clone the repo
bashgit clone https://github.com/kinchitgupta/smart-healthcare-ai.git
cd smart-healthcare-ai
Step 2 — Create virtual environment
bashpython -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
Step 3 — Install dependencies
bashpip install -r backend/requirements.txt
Step 4 — Import doctors into database (run once)
bashpython import_doctors.py
Step 5 — Start the backend
bashcd backend
uvicorn main:app --reload --port 8001
Step 6 — Open frontend
Open any of these files directly in your browser:

🔁 Demo Flow
1. Open nurse.html
2. Register a new patient
3. Enter critical vitals (high HR, low SpO₂, chest pain ✓, heart disease ✓)
4. Click "Run AI Triage"
5. Note the assigned doctor credentials shown
6. Open doctor_login.html → login with those credentials
7. See patient appear on the dashboard with severity badge
8. Send instructions or book appointment
9. Patient checks status on status.html
