# 🏥 Smart Healthcare AI — Intelligent Triage & Decision Support System

> Built by me for a Hackathon | B.Tech Final Year Project | 2025–26

I designed and built this end-to-end AI-powered hospital triage system myself — from the ML model to the backend to the frontend. It analyzes patient vitals in real time, predicts severity using a model I trained, and routes patients to the right doctor — all in under 2 seconds.

---

## 🎯 What It Does

Hospitals waste critical minutes manually triaging patients. I built this system to automate that:

1. Nurse enters patient vitals
2. My AI model predicts severity — **GREEN / YELLOW / ORANGE / RED**
3. System auto-assigns the right specialist doctor
4. Doctor reviews patients on a priority dashboard
5. Patient checks their own status and receives doctor instructions

---

## 🤖 AI Model (Trained by Me)

| Metric | Score |
|--------|-------|
| Model | Gradient Boosting Classifier |
| Accuracy | ~95% |
| ROC-AUC | 0.97 |
| Training Data | 2000 patient records |
| Features | 29 engineered features |

I engineered the features myself, including: Heart Rate, SpO₂, Blood Pressure, Temperature, Respiratory Rate, Chest Pain, Diabetes, Heart Disease history, plus custom-engineered ratios (Shock Index, HR/Oxygen ratio, Danger Score, etc.)

## ⚙️ Setup & Run

### Prerequisites
- Python 3.10+
- MySQL running locally
- MySQL database named `ai_triage_db`

### Step 1 — Clone the repo
```bash
git clone https://github.com/kinchitgupta/smart-healthcare-ai.git
cd smart-healthcare-ai
```

### Step 2 — Create virtual environment
```bash
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux
```

### Step 3 — Install dependencies
```bash
pip install -r backend/requirements.txt
```

### Step 4 — Import doctors into database (run once)
```bash
python import_doctors.py
```

### Step 5 — Start the backend
```bash
cd backend
uvicorn main:app --reload --port 8001
```

### Step 6 — Open frontend
Open any of these files directly in your browser:

| File | Purpose |
|------|---------|
| `frontend/index.html` | Landing page |
| `frontend/nurse.html` | Nurse station |
| `frontend/doctor_login.html` | Doctor login |
| `frontend/doctor.html` | Doctor dashboard |
| `frontend/status.html` | Patient status check |

## 🌐 API Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/patient/register` | Register new patient |
| POST | `/api/nurse/vitals` | Submit vitals + run AI triage |
| POST | `/api/doctor/login` | Doctor authentication |
| GET | `/api/doctor/patients` | Get patient queue |
| POST | `/api/doctor/recommend` | Save doctor notes |
| POST | `/api/doctor/send-message` | Message patient |
| GET | `/api/patient/status/{id}` | Patient status check |
| POST | `/api/patient/upload-doc` | Upload report/document |
| GET | `/api/stats` | System statistics |

Full API docs available at: `http://localhost:8001/docs` (Swagger UI auto-generated by FastAPI)

## 🧠 Severity Levels

| Level | Meaning | Action |
|-------|---------|--------|
| 🟢 GREEN | Safe — normal vitals | No immediate action |
| 🟡 YELLOW | Moderate | Consult doctor within 2 hours |
| 🟠 ORANGE | Urgent | Specialist consultation today |
| 🔴 RED | Critical | Immediate emergency intervention |
